What are Attributes ?

Attributes in the context of a database refer to the properties or characteristics that describe entities (objects or concepts) within an entity set. Attributes provide detailed information about the entities and are used to store, retrieve, and manipulate data.

Here are the main types of attributes listed below :-

  • Simple and Composite Attributes
  • Single-Valued and Multi-Valued Attributes
  • Stored and Derived Attributes
  • Complex Attributes
  • Key Attributes

Simple and Composite Attributes

  • Simple Attribute
      • A simple attribute is an attribute that cannot be divided any further into meaningful components.
      • It represents an atomic, indivisible value associated with an entity.
      • Example: In a "Consumer" entity set, "CPI" is a simple attribute.
  • Composite Attribute
      • A composite attribute is an attribute that can be divided into sub-parts, each with its own meaning.
      • It represents a structured value composed of multiple components.
      • Example: In a "Name" attribute, you can have sub-components like "First Name", "Last Name" and "Middle Name".

Single-Valued and Multi-Valued Attributes

  • Single-Valued Attribute
      • A single-valued attribute holds a single value for each entity.
      • It does not allow multiple values or a list of values.
      • Example: "Birthdate" is a single-valued attribute for a "Person."
  • Multi-Valued Attribute
      • A multi-valued attribute can hold multiple values or a list of values for each entity.
      • Example: A "EmailID" attribute for a "Contact" entity could be multi-valued because a person may have multiple Email Addresses.

Stored and Derived Attributes

  • Stored Attribute
      • A stored attribute is one whose value is explicitly stored in the database.
      • It is part of the database schema and is typically retrieved directly from the database.
      • Example: "Height" of a "Person" entity is a stored attribute.
  • Derived Attributes
      • A derived attribute is not explicitly stored in the database but can be computed or derived from other attributes.
      • Example: "Age" of a "Person" can be derived from the "DateOfBirth" attribute by subtracting the birthdate from the current date.

Complex Attributes

  • Complex Attributes
      • A complex attribute is an attribute that can have its own attributes (sub-attributes) within it.
      • It is a combination of simple or composite attributes.
      • Example: An "Name" attribute may have sub-attributes like "First Name", "Middle Name" and "Last Name" each of which can be simple or composite attributes themselves.

Key Attributes

  • Key Attribute
      • A key attribute is an attribute that is used to uniquely identify instances of an entity within an entity set.
      • It ensures that no two entities in the same set have the same value for this attribute.
      • Example: In a "Student" entity set, "Roll_No" can be a key attribute, ensuring that each student has a unique identifier.

Descriptive Attribute

    • If any relationship has a attribute like entity then its known as Descriptive Attributes.
    • Example: Student had been issued degree certificate on 25/5/2020.Certificate date is an attribute of relationship.
