What are Functions ?

In MySQL, functions are predefined routines or procedures that perform specific tasks or calculations on data. Functions take one or more input values (arguments), process them, and return a result. Functions are an integral part of SQL and are used to perform various operations on data within SQL queries.

In this page we will be studying the following Functions :-

  • Aggregate Functions
  • Character Functions
  • Arithmetic Functions
  • Date Functions
  • Conversion Functions

What are Aggregate Functions ?

Aggregate functions in MySQL are a category of functions that operate on sets of values and return a single value as a summary or result. These functions perform calculations on multiple rows or records, typically within a specified group, and return a single value that summarizes the data in some way.

Here are the main Aggregate Functions :-

  • SUM
  • AVG
  • COUNT
  • MAX
  • MIN

    So let's take an reference table named "Employees" to understand all the above Function :-

    Emp_Id Emp_First_Name Emp_Last_Name Department Salary
    1 Aryansh Rajput Manager 80000
    2 Kush Patel HR 78000
    3 Shivanand Singh Finance 75000
    4 Ishant Rathi Developer 70000

    SUM Function

    SUM :- Calculates the sum of a numeric column's values.

    SUM(column_name)

    Now let's see how a query is written using SUM function :-

    SELECT SUM(Salary) AS Total_Salary FROM Employees;

    So the output of the above code will be :-

    Total_Salary
    303000

    AVG Function

    AVG :- Calculates the average (mean) of a numeric column's values.

    AVG(column_name)

    Now let's see how a query is written using AVG function :-

    SELECT AVG(salary) AS Average_salary FROM Employees;

    So the output of the above code will be :-

    Average_Salary
    75750

    COUNT Function

    COUNT :- Counts the number of rows that match a condition or the number of non-null values in a column.

    COUNT(*)

    Now let's see how a query is written using COUNT function :-

    SELECT COUNT(*) AS Total_Employees FROM Employees;

    So the output of the above code will be :-

    Total_Employees
    4

    MAX Function

    MAX :- Returns the maximum value from a numeric or date column.

    MAX(column_name)

    Now let's see how a query is written using MAX function :-

    SELECT MAX(Salary) AS Highest_Salary FROM Employees;

    So the output of the above code will be :-

    Highest_Salary
    80000

    MIN Function

    MIN :- Returns the minimum value from a numeric or date column.

    MIN(column_name)

    Now let's see how a query is written using MIN function :-

    SELECT MIN(Salary) AS Lowest_Salary FROM Employees;

    So the output of the above code will be :-

    Lowest_Salary
    70000

What are Character Function?

Character functions in MySQL are a category of functions that are used to manipulate and modify text or character data within SQL queries. These functions allow you to perform various operations on character strings, such as concatenation, conversion, case conversion (uppercase and lowercase), and substring extraction.

Here are the main Character Functions :-

  • CONCAT
  • UPPER
  • LOWER
  • LENGTH
  • SUBSTRING
  • CONVERT
  • TRIM
  • LEFT AND RIGHT

    CONCAT Function

    CONCAT :- Concatenates two or more strings together.

    CONCAT(column1,' ',column2)

    So let's take a reference table to understand the following CONCAT function :-

    Emp_Id Emp_First_Name Emp_Last_Name Department Salary
    1 Aryansh Rajput Manager 80000
    2 Kush Patel HR 78000
    3 Shivanand Singh Finance 75000
    4 Ishant Rathi Developer 70000

    Now let's see how a query is written using CONCAT function :-

    SELECT CONCAT(emp_first_name,' ', emp_last_name) AS Full_Name FROM Employees;

    So the output of the above code will be :-

    Full_Name
    Aryansh Rajput
    Kush Patel
    Shivanand Singh
    Ishant Rathi

    UPPER Function

    UPPER :- Converts a string to uppercase.

    UPPER(column_name)

    So let's take an reference table named "Employees" to understand the CONCAT Function :-

    Emp_Id Emp_Name Department Salary
    1 Aryansh Rajput Manager 80000
    2 Kush Patel HR 78000
    3 Shivanand Singh Finance 75000
    4 Ishant Rathi Developer 70000

    Now let's see how a query is written using UPPER function :-

    SELECT UPPER(Emp_Name) AS uppercase_name FROM Employees;

    So the output of the above code will be :-

    Uppercase_Name
    ARYANSH RAJPUT
    KUSH PATEL
    SHIVANAND SINGH
    ISHANT RATHI

    LOWER Function

    LOWER :- Converts a string to lowercase.

    LOWER(column_name)

    So let's take an reference table named "Employees" to understand the LOWER Function :-

    Emp_Id Emp_Name Department Salary
    1 Aryansh Rajput Manager 80000
    2 Kush Patel HR 78000
    3 Shivanand Singh Finance 75000
    4 Ishant Rathi Developer 70000

    Now let's see how a query is written using LOWER function :-

    SELECT LOWER(Emp_Name) AS uppercase_name FROM Employees;

    So the output of the above code will be :-

    Lowercase_Name
    aryansh rajput
    kush patel
    shivanand singh
    ishant rathi

    LENGTH Function

    LENGTH :- Returns the length (number of characters) of a string.

    LENGTH(column_name)

    So let's take an reference table named "Employees" to understand LENGTH Function :-

    Emp_Id Emp_First_Name Emp_Last_Name Department Salary

    3 Shivanand Singh Finance 75000

    Kush 4
    Shivanand 9
    Ishant 6

    SUBSTRING Function

    SUBSTRING :- Extracts a portion of a string based on a starting position and length.

    SUBSTRING(column_name)

    So let's take an reference table named "Products" to understand the SUBSTRING Function :-

    Product_Id Product_Name
    1 Smartphone: iPhone 13 Pro Max
    2 Laptop: Dell XPS 15
    3 Tablet: Samsung Galaxy Tab S7
    4 Headphones: Sony WH-1000XM4

    Now let's see how a query is written using SUBSTRING function :-

    SELECT product_name, SUBSTRING(product_name, 1, 10) AS Short_Name FROM Products;

    So the output of the above code will be :-

    Product _Name Short_Name
    Smartphone: iPhone 13 Pro Max Smartphone
    Laptop: Dell XPS 15 Laptop: De
    Tablet: Samsung Galaxy Tab S7 Tablet: Sa
    Headphones: Sony WH-1000XM4 Headphones

    CONVERT Function

    CONVERT :- Converts a string from one character set to another.

    CONVERT(column_name, Data_Type)

    So to understand the CONVERT function we are going to take a reference table named Employees :-

    Emp_Id Emp_Name Salary
    1 Aryansh Rajput 62000.25
    2 Kush Patel 60000.75
    3 Shivanand Singh 58000.00
    4 Ishant Rathi 55000.50

    Note :- So in the above code the Salary column is having VARCHAR data type and now using CONVERT function we are going to change it's data type from "VARCHAR" to "DECIMAL". And if you want to check the datatype of a specific column the Query is as follows :-

    desc Employees;

    So the output of the above code will be :-

    Field Type NULL Key Default Extra
    Emp_Id varchar(20) YES -- NULL --
    Emp_Name varchar(30) YES -- NULL --
    Salary varchar(15) YES -- NULL -- 
    SELECT Emp_Name, CONVERT(Salary, DECIMAL(10, 2)) AS Converted_Salary FROM Employees;
    Emp_Name Salary
    Aryansh Rajput 62000.25
    Kush Patel 60000.75
    Shivanand Singh 58000.00
    Ishant Rathi 55000.50

    TRIM Function

    TRIM :- Removes leading and trailing spaces (or specified characters) from a string.

    TRIM(' ' FROM column_name)

    So let's take an reference table named "Employees" to understand the TRIM Function :-

    Emp_Id Emp_Name Department Salary
    1 Aryansh Rajput Manager 80000
    2 Kush Patel HR 78000
    3 Shivanand Singh Finance 75000
    4 Ishant Rathi Developer 70000

    Now let's see how a query is written using TRIM function :-

    SELECT TRIM(' ' FROM Emp_Name) AS Trimmed_Name FROM Employees;

    So the output of the above code will be :-

    Trimmed_Name
    AryanshRajput
    KushPatel
    ShivanandSingh
    IshantRathi

    We can also use TRIM function to delete specific character from a specific column and the Query for that is as follows :-

    • To Delete The Leading Element :-
      • SELECT TRIM(LEADING 'X' FROM 'XXDataXX') AS Trimmed_String;

      The output of the above code is as follows :-

      Trimmed_String
      DataXX

    • To Delete The Trailing Element :-
      • SELECT TRIM(TRAILING 'X' FROM 'XXDataXX') AS trimmed_string;

      The output of the above code is as follows :-

      Trimmed_String
      XXData

    LEFT and RIGHT

    LEFT and RIGHT :- Extracts a specified number of characters from the left or right side of a string.

    • LEFT :-
      • LEFT(column_name, range)

      So let's take a reference table to understand the following LEFT function :-

      First_Four_Letters
      Arya
      Kush
      Shiv
      Isha

    • RIGHT :-
      • RIGHT(column_name, range)

      So let's take a reference table to understand the following RIGHT function :-

      Emp_Id Emp_First_Name Emp_Last_Name Department Salary
      1 Aryansh Rajput Manager 80000
      2 Kush Patel HR 78000
      3 Shivanand Singh Finance 75000
      4 Ishant Rathi Developer 70000

      Now let's see how a query is written using RIGHT function :-

      Kush
      nand
      hant

What are Arithmetic Functions ?

Arithmetic functions in MySQL are a category of functions used to perform mathematical operations on numeric data. These functions enable you to conduct calculations on numeric values within your SQL queries.

Here are the main Arithmetic Functions :-

  • ABS
  • ROUND
  • CEIL
  • FLOOR
  • MOD
  • POWER
  • SQRT
  • SIGN

    ABS Function

    ABS :- Returns the absolute value of a number. It removes the negative sign if present.

    ABS(column_name)

    So let's take an reference table named "numbers" to understand the ABS Function :-

    Value
    -10
    7.5
    -5.5

    Now let's see how a query is written using ABS function :-

    SELECT Value, ABS(Value) AS Absolute_Value FROM numbers;

    So the output of the above code will be :-

    Value Absolute_Value
    -10 10
    7.5 7.5
    -5.5 5.5

    ROUND Function

    ROUND :- Rounds a number to a specified number of decimal places.

    ROUND(column_name)

    So let's take an reference table named "numbers" to understand the ROUND Function :-

    Value
    -7.4867
    8.6814
    9.8743

    Now let's see how a query is written using ROUND function :-

    SELECT Value, ROUND(Value, 3) AS Rounded_Value FROM numbers;

    So the output of the above code will be :-

    Value Rounded_Value
    -7.4867 -7.487
    8.6814 8.681
    9.8743 9.874

    CEIL Function

    CEIL :- Rounds a number up to the nearest integer that is greater than or equal to the original number.

    CEIL(column_name)

    So let's take an reference table named "numbers" to understand the CEIL Function :-

    Value
    -7.4867
    8.6814
    9.8743

    Now let's see how a query is written using CEIL function :-

    SELECT Value, CEIL(Value) AS Ceiling_Value FROM numbers;

    So the output of the above code will be :-

    Value Ceiling_Value
    -7.4867 -7
    8.6814 9
    9.8743 10

    FLOOR Function

    FLOOR :- Rounds a number down to the nearest integer that is less than or equal to the original number.

    FLOOR(column_name)

    So let's take an reference table named "numbers" to understand the FLOOR Function :-

    Value
    -7.4867
    8.6814
    9.8743

    Now let's see how a query is written using FLOOR function :-

    SELECT Value, FLOOR(Value) AS Floor_Value FROM numbers;

    So the output of the above code will be :-

    Value Floor_Value
    -7.4867 -8
    8.6814 8
    9.8743 9

    MOD Function

    MOD :- Returns the remainder of the division of one number by another.

    MOD(column_name, divisor)

    So let's take two reference table named "numbers" and "divisors" to understand the MOD Function :-

    • Number table :-
      • Value
      -10
      7.5
      -5.5

    • Divisors table :-
      • Divisor
      4
      3


    Now let's see how a query is written using MOD function :-

    SELECT n.Value, d.Divisor, MOD(n.Value, d.Divisor) AS Remainder
    FROM numbers n
    CROSS JOIN divisors d;

    So the output of the above code will be :-

    Value Divisor Remainder
    -10 3 -1
    -10 4 -2
    7.25 3 1.25
    7.25 4 3.25
    -5.5 3 -2.5
    -5.5 4 -1.5

    POWER Function

    POWER :- Raises a number to a specified power.

    POWER(column_name, exponent)

    So let's take two reference table named "numbers" and "Exponents" to understand the MOD Function :-

  1. Number table :-
    2. Value
    -10
    7.5
    -5.5

  2. Exponents table :-
    3. Exponent
    2
    3


    Now let's see how a query is written using POWER function :-

    SELECT Value, e.Exponent, POWER(Value, e.Exponent) AS Result
    FROM numbers
    CROSS JOIN exponents e;
    Value Exponent Result
    -10 3 -1000
    -10 2 100
    7.25 3 381.078125
    7.25 2 52.5625
    -5.5 3 -166.375
    -5.5 2 30.25

    SQRT Function

    SQRT :- Calculates the square root of a number.

    SQRT(column_name)

    So let's take an reference table named "numbers" to understand the SQRT Function :-

    Value
    -10
    6.25
    5.5

    Now let's see how a query is written using SQRT function :-

    SELECT value, SQRT(value) AS Square_Root FROM numbers;

    So the output of the above code will be :-

    Value Square_Root
    -10 NULL
    6.25 2.5
    5.5 2.345207879911715

    SIGN Function

    SIGN :- Returns the sign of a number as -1 for negative, 0 for zero, and 1 for positive.

    SIGN(column_name)

    So let's take an reference table named "numbers" to understand the SIGN Function :-

    Value
    -10
    7.5
    -5.5

    Now let's see how a query is written using SIGN function :-

    SELECT Value, SIGN(Value) AS Sign_Value FROM numbers;

    So the output of the above code will be :-

    Value Sign_Value
    -10 -1
    7.5 1
    -5.5 -1

  • DATE

  • MONTH

  • MINUTE
  • SECOND
  • DATE_FORMAT

  • DATE_ADD

What are Conversion Functions ?

Conversion functions in MySQL are used to convert data from one data type to another. They are essential for ensuring that data is in the correct format for operations, comparisons, or display purposes. MySQL provides various conversion functions to help you transform data from one type to another.

Here are the main Conversion Functions :-

  • CAST
  • CONVERT
  • CONVERT_TZ
  • BINARY OPERATOR
  • HEX
  • UNHEX
  • JSON

    CAST Functions

    CAST() :- Converts a value from one data type to another specified data type.

    CAST(column_name AS type)

    SO let's take a reference table named "Data" having a column named "Value" containing Numeric values as string :-

    Value
    '42'
    '3.14'
    '100'

    Now let's see how a query is written using CAST function :-

    SELECT Value, CAST(Value AS SIGNED) AS Numeric_Value FROM Data;
    Value Numeric_Value
    '42' 42
    '3.14' 3
    '100' 100

    CONVERT Functions

    CONVERT() :- Converts a value from one character set to another character set.

    CONVERT(column_name USING type)

    SO let's take a reference table named "Text_Data" having a column named "Text" containing Numeric values as string :-

    Text
    Hello, world!
    mlaut

    Now let's see how a query is written using CONVERT function :-

    SELECT Text, CONVERT(text USING BINARY) AS Converted_Text FROM Text_Data;
    Text Converted_Text
    Hello, world! 0x48656C6C6F2C20776F726C6421
    mlaut 0xC3BC6D6C617574

    CONVERT_TZ Functions

    CONVERT_TZ() :- Converts a datetime value from one time zone to another.

    CONVERT_TZ(column_name, UTC, AREA)

    SO let's take a reference table named "Events" having a column named "Event_Datetime" containing Numeric values as string :-

    Event_Datetime
    2023-08-15 14:30:00
    2023-09-20 09:45:00

    Now let's see how a query is written using CONVERT_TZ function :-

    SELECT event_datetime, CONVERT_TZ(event_datetime, 'UTC', 'America/New_York') AS Converted_Datetime FROM events;
    Event_Datetime Converted_Datetime
    2023-08-15 14:30:00 2023-08-15 10:30:00
    2023-09-20 09:45:00 2023-09-20 05:45:00

    BINARY Functions

    BINARY Operator :- Converts a string to a binary string (non-binary to binary).

    BINARY column_name

    SO let's take a reference table named "Text_Data" having a column named "Text" containing Numeric values as string :-

    Text
    Hello, world!
    mlaut

    Now let's see how a query is written using BINARY function :-

    SELECT Text, BINARY Text AS Binary_Value FROM Text_Data;
    Text Converted_Text
    Hello, world! 0x48656C6C6F2C20776F726C6421
    mlaut 0xC3BC6D6C617574

    HEX Functions

    HEX() :- Converts a string to its hexadecimal representation.

    HEX(column_name)

    SO let's take a reference table named "Text_Data" having a column named "Text" containing Numeric values as string :-

    Text
    Hello, world!
    mlaut

    Now let's see how a query is written using HEX function :-

    SELECT Text, HEX(Text) AS Hexadecimal_Value FROM Text_Data;
    Text Converted_Text
    Hello, world! 48656C6C6F2C20776F726C6421
    mlaut C3BC6D6C617574

    UNHEX Functions

    UNHEX() :- Converts a hexadecimal string to its original string value.

    UNHEX(column_name)

    SO let's take a reference table named "Unhex_Data" having a column named "Hex_Value" containing Numeric values as string :-

    Hex_Value
