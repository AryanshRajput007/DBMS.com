What is Mapping Cardinality ?

In the context of database design, "mapping cardinality" (also known as "relationship cardinality" or "participation cardinality") describes the number of instances in one entity set (table) that are associated with the number of instances in another entity set through a relationship. It defines the nature and constraints of the relationships between entities in an Entity-Relationship Diagram (ERD) or relational database schema.

Here are the main types if Mapping Cardinality listed below :-

  • One-to-One Relationship (1:1)
  • One-to-Many Relationship (1:n)
  • Many-to-One Relationship (n:1)
  • Many-to-Many Relationship (n:m)

One-to-One Relationship (1:1)

  • In a one-to-one relationship, each entity in one entity set is associated with exactly one entity in another entity set, and vice versa.
  • This type of relationship is relatively rare because it implies a strict one-to-one correspondence between two entities.

One-to-Many Relationship

  • In a one-to-many relationship, each entity in one entity set is associated with one or more entities in another entity set, but each entity in the second set is associated with only one entity in the first set.
  • This type of relationship is common in many real-world scenarios.
  • Example :- In the below example a single customer is having multiple ongoing loans.

Many-to-One Relationship

  • In a many-to-one relationship, multiple entities in one entity set can be associated with a single entity in another entity set, but each entity in the second set is associated with only one entity in the first set.
  • This type of relationship is common when modeling hierarchies or categorizations.
  • Example :- In the below example multiple students have enrolled for a single course.

Many-to-Many Relationship

  • In a many-to-many relationship, multiple entities in one entity set can be associated with multiple entities in another entity set, and vice versa.
  • This type of relationship is used when entities from both sets can have multiple associations with each other.
  • Example :- In the below example multiple students have taken multiple courses.
